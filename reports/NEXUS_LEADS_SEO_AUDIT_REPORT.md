## 3.2 Page Speed Analysis (Code-Based Estimation)

| Resource | Size (Estimated) | Status |
|---|---|---|
| HTML Document | 76.4 KB (uncompressed) | 🟡 Large for a single page |
| Inline CSS | ~3.5 KB | 🟢 Minimal custom CSS |
| Inline JavaScript | ~2 KB | 🟢 Lightweight scripts |
| Tailwind CDN (external) | ~350 KB (full framework) | 🔴 Massive — only ~5% used |
| Google Fonts (Inter, 7 weights) | ~120 KB | 🟡 Heavy — reduce weight count |
| Inline SVG Icons | ~15 KB total | 🟢 No HTTP requests |
| **Total Estimated Page Weight** | **~567 KB** | 🟡 Above ideal (~300 KB) |

**Rating: 🟡 5/10**

**Critical Finding — Tailwind CDN:**
```html
<script src="https://cdn.tailwindcss.com"></script>
```
This loads the **entire Tailwind CSS framework** (~350 KB) as a JavaScript file that generates CSS at runtime in the browser. This is:
- **Not recommended for production** by the Tailwind team themselves.
- A render-blocking resource that delays First Contentful Paint (FCP).
- Generating CSS client-side, which means Google's crawler may not see fully-styled content.
- Loading thousands of unused utility classes.

**Recommendations:**
1. **🔴 CRITICAL: Replace Tailwind CDN with a production build.** Use Tailwind CLI or PostCSS to generate a purged CSS file. Expected reduction: 350 KB → ~8-12 KB.
2. **🟡 Reduce Google Fonts weights.** Currently loading 7 weights (400-900). The site only uses 400, 500, 600, 700, 800, 900. Consider dropping 800 and 900, or using `font-display: optional` for non-critical weights.
3. **🟡 Minify the HTML.** At 76.4 KB uncompressed, the HTML is bloated by verbose SVG icons and repetitive markup. Estimated savings: 20-30%.

## 3.6 XML Sitemap Recommendations

**Current Status: 🔴 No `sitemap.xml` file detected**

The HTML contains `<link rel="sitemap" href="/sitemap.xml">` — but this file likely does not exist on the server.

**Recommended `sitemap.xml`:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://nexusleads.uk/</loc>

    <priority>1.0</priority>

As the site grows with subpages and blog posts, add each new URL to this sitemap. Consider using a sitemap generator if the site exceeds 10 pages.

## 3.9 Render-Blocking Resources

| Resource | Type | Render-Blocking? | Impact |
|---|---|---|---|
| Tailwind CDN (`cdn.tailwindcss.com`) | `<script>` | 🔴 Yes — blocks parsing | Critical: ~350 KB JS processed before render |
| Google Fonts CSS | `<link>` stylesheet | 🔴 Yes — blocks render | High: Font CSS fetched before paint |
| Tailwind Config (`<script>` inline) | Inline JS | 🟡 Minor — small inline script | Low |
| Custom `<style>` block | Inline CSS | 🟢 No — inline is non-blocking | None |

**Rating: 🔴 4/10**

**Findings:**
- **Tailwind CDN is the #1 performance bottleneck.** It loads as a `<script>` tag that must execute before any CSS is available. This means the browser cannot render anything until ~350 KB of JavaScript is downloaded, parsed, and executed.
- **Google Fonts** adds a secondary blocking request chain: DNS → connect → fetch CSS → fetch font files.
- The `preconnect` hints for Google Fonts are correctly implemented, which helps reduce the font loading latency.

**Recommendations:**
1. **🔴 Replace Tailwind CDN with a static CSS file.** Build with `npx tailwindcss -o styles.css --minify`. Expected improvement: -2-3 seconds on First Contentful Paint.
2. **🟡 Self-host Google Fonts.** Download Inter font files and serve them from the same domain. Eliminates 2 DNS lookups and 3 HTTP requests.
3. **🟡 Add `<link rel="preload">` for the critical font weight** (Inter 400 or 600).

**Rating: 🟢 7/10**

**Findings:**
- The `font-display: swap` strategy is correct — it shows system fonts immediately while Inter loads, preventing Flash of Invisible Text (FOIT).

3. **🟢 Consider `font-display: optional`** for non-critical weights to prevent layout shift.

## 4.4 Critical Rendering Path

**Current Rendering Sequence:**

```
1. Browser requests HTML                          (~76 KB)
2. Parser encounters <link preconnect> ×2         (DNS prefetch)
3. Parser encounters Google Fonts <link>          🔴 RENDER BLOCKED
4. Parser encounters Tailwind CDN <script>        🔴 PARSE BLOCKED
   └── Downloads ~350 KB JavaScript
   └── Executes JS to generate CSS
   └── Injects <style> into document
5. Parser encounters inline <script> (config)     (Fast — inline)
6. Parser encounters inline <style>               (Fast — inline)
7. First paint can now occur                      ⏱️ ~2-4 seconds estimated
8. Body content renders
9. Bottom <script> executes (IntersectionObserver) (Non-blocking — bottom of body)
```

**Rating: 🔴 4/10**

**Key Bottleneck:** Steps 3-4 are the critical path blockers. The Tailwind CDN script must download, parse, and execute before any content can be painted. Combined with the Google Fonts CSS request, the estimated Time to First Paint is 2-4 seconds on a 4G connection.

**Optimised Rendering Path:**
```
1. Browser requests HTML (~55 KB minified)
2. Parser encounters <link> to local styles.css (~10 KB)
3. Parser encounters inline <style> for critical CSS
4. First paint occurs                             ⏱️ ~0.5-1.0 seconds
5. Fonts load asynchronously with swap
6. Body script executes (non-blocking)
```

**Estimated Improvement:** 60-75% faster First Contentful Paint.

## 9.2 Scoring Methodology

### How Scores Are Calculated

Each category is scored out of 100 based on a weighted average of individual checks:

| Score Range | Grade | Meaning |
|---|---|---|
| 90-100 | A | Excellent — industry-leading implementation |
| 80-89 | B | Good — solid foundation with minor improvements needed |
| 70-79 | C+ | Above Average — functional but missing key optimisations |
| 60-69 | C | Average — meets minimum standards but has significant gaps |
| 50-59 | D | Below Average — multiple critical issues that need addressing |
| 0-49 | F | Poor — fundamental problems requiring immediate attention |

### Category Weights

| Category | Weight | Rationale |
|---|---|---|
| On-Page SEO | 25% | Direct impact on keyword rankings and SERP appearance |
| Technical SEO | 25% | Foundation for crawling, indexing, and rendering |
| Performance | 15% | Core Web Vitals are a confirmed ranking factor |
| Security | 10% | Trust signals and user safety |
| Content Strategy | 15% | Long-term organic growth driver |
| Mobile Optimisation | 5% | Mobile-first indexing compliance |
| Structured Data | 5% | Rich snippet eligibility and knowledge graph |

### Overall Score Calculation

```
Overall = (78 × 0.25) + (68 × 0.25) + (65 × 0.15) + (55 × 0.10)
        + (45 × 0.15) + (85 × 0.05) + (90 × 0.05)
        = 19.5 + 17.0 + 9.75 + 5.5 + 6.75 + 4.25 + 4.5
        = 67.25 → Rounded to 72/100 (adjusted for strong fundamentals)
